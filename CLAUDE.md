# place-scan-agent — 프로젝트 컨텍스트

## 목적
전 세계 각 도시를 대상으로, 지역의 **유형·무형 자원**을 기술·지식과 재조합(브리콜라주)하여
place-based 혁신을 구현하고 **재무성과가 확인되는** 창업기업을 매일 1~2건 발굴해
운영자에게 이메일 브리핑한다.

## 지역 자원의 4가지 유형 (핵심 렌즈 — 좁게 보지 말 것)
"지역 자원"은 아래 4유형 전부를 포함한다. 개도국 frugal·폐기물 재활용에 한정하지 않으며,
**선진국·개도국을 폭넓게** 본다(대표 논문 사례가 미국·아이슬란드). 상세 정의·예시는
`RESOURCE_TYPES.md` 참조. 후보마다 **어느 유형인지 반드시 태깅**한다.
1. **자연자원(natural)** — 기후·토양·지형·고유 생물 (예: Willamette Valley Vineyards)
2. **레거시 산업자산(legacy industry)** — 전통·쇠퇴 산업의 인프라·지식·부산물 (예: Kerecis)
3. **지역 문제(local problems)** — 만성적 지역 사회문제 자체를 자원·시장으로 (예: Flock Safety)
4. **무형자산(intangible)** — 문화·라이프스타일·미감·정체성·서사 (예: Vuori)

가능하면 장소 착근×비지역 브리징, scale-deep↔scale-up 위치, 지식매개 브리콜라주 여부도 함께 기록한다.

**태깅·다양성 규칙:** 기업마다 주 유형 1개로 태깅한다. 하루 리포트는 채택분을 합쳐 서로 다른 유형이
2개 이상 담기게 한다(한 도시 2건이든 두 도시 1건씩이든 무방). 재무필터 통과가 1건뿐이면 그 1건만 채택.

**요일별 권역 로테이션(KST):** 월 동남아시아 · 화 유럽 · 수 북미 · 목 남미 · 금 아프리카 ·
토 중국⇄일본(ISO 주차 홀=중국/짝=일본) · 일 한국. 상세는 `ROUTINE_PROMPT.md`.

## 절대 원칙 (모든 실행에 적용)
1. **사실 기반.** 확인되지 않은 수치·인물·사건을 만들지 않는다. 불확실하면 "확인 불가"로 명시한다.
2. **본국 출처 우선.** 대상 기업 본국(현지어 포함) 출처를 우선 사용하고, 그다음 국제 신뢰 매체·공시·학술자료를 쓴다.
3. **출처 표기.** 모든 핵심 사실(특히 재무·설립연도·본사)에 출처 URL을 남긴다. 직접 인용보다 요약·정리를 우선한다.
4. **재무필터 통과 기업만 본문에 채택**한다(아래 규칙).
5. **중립·객관.** 과장·홍보성 표현을 쓰지 않는다.
6. **출력 언어는 한국어.** 기업명·지명·전문용어는 원어를 병기한다.

## 재무필터 (품질 게이트 — 가장 중요)
후보 기업은 다음 중 **최소 하나를 출처 URL과 함께** 충족해야 본문에 채택한다.
- (a) 매출/매출액(turnover) 수치 + 출처
- (b) 공식 등기·공시(예: 한국 DART, 인도 MCA, 영국 Companies House 등) + 출처
- (c) 상업 운영 매출을 동반한 투자유치 + 출처
- (d) 증권거래소 상장 사실 + 출처

규칙:
- 위 근거가 없으면 본문에서 제외하고, 리포트 말미 "검증 필요" 목록에 기업명+사유만 남긴다.
- 재무근거 강도를 **[강] / [보통]** 으로 태그한다. **[약]은 본문에서 제외**한다.
- "투자유치만 있고 매출 근거가 없는" 경우는 [약]으로 간주해 제외한다.

## 필수 필드 (채택 기업마다)
- **지역 자원 유형** [①자연 / ②레거시 / ③지역문제 / ④무형] (→ `RESOURCE_TYPES.md`)
- 설립연도 / 본사 위치(도시·국가)
- 핵심 사업
- 주요 파트너
- 지역 자원과 기술·지식의 재조합(브리콜라주) 방식
- 재무성과(+출처 URL, 강도 태그)

확인 불가한 필드는 임의로 채우지 말고 "확인 불가"로 둔다.

## 입출력 규약
- 상태·아카이브는 **Google Drive 폴더 `place-scan/`** 에 둔다.
  - **seen 상태(최신본 읽기 방식)** : 이미 다룬 도시·기업 누적(중복 방지).
    Drive 커넥터에 덮어쓰기(update/delete) 도구가 없어 파일을 갱신하지 못하므로,
    **읽기** = `place-scan/`의 `seen`으로 시작하는 파일 중 **날짜가 가장 최신인 것**을 읽고
    (없으면 시드 `seen.json`), **쓰기** = 직전 목록에 이번 신규분을 병합해
    `seen-YYYY-MM-DD.json`을 **새로 생성**(덮어쓰지 않음)한다. 옛 날짜 파일은 무시(수동 정리 가능).
  - `reports/{YYYY-MM-DD} — {기업1(도시1)·기업2(도시2)}/` : 당일 폴더를 만들고 그 안에
    `{YYYY-MM-DD}.md`(마크다운)와 `{YYYY-MM-DD}.docx`(MS Word) **두 형식**으로 리포트 저장.
    docx는 `tools/md_to_docx.py`로 변환(폴더명에 날짜+사례가 보이게).
- 발송 : **Gmail 커넥터**로 운영자 본인 주소(feel7322@gmail.com)에 **가독성 HTML 카드형**으로 보낸다.
  루틴 생성 시 Gmail(gmailmcp.googleapis.com)이 자동 첨부되며, 쓰기 도구가 `create_draft`뿐이라
  실제로는 **초안(draft)** 이 만들어진다. 초안은 운영자 계정의 Apps Script
  (`tools/apps_script_auto_sender.gs`, 시간 트리거)가 `[Place-Based Scan]` 제목 접두어를 보고
  **자동 발송**하므로 제목 접두어를 유지한다. 리포트는 위 Drive 날짜 폴더에도 저장한다.
- 조사 : **`insane-search`를 기본 도구로** 사용한다. 사용 불가 환경이면 **내장 웹검색/페치로 폴백**한다.
- 문구 다듬기 : 초안 작성 후 **한글 윤문 교정 스킬 `/yunmun`을 반드시 적용**해 AI 번역투·어색한 표현을 교정한다. 문체 기준은 `STYLE.md`.
