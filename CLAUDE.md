# place-scan-agent — 프로젝트 컨텍스트

## 목적
전 세계 각 도시를 대상으로, 지역의 **유형·무형 자원**을 기술과 융합하여
place-based / 브리콜라주(bricolage) / frugal 혁신을 구현하고
**재무성과가 확인되는** 창업기업을 매일 1~2건 발굴해 운영자에게 이메일 브리핑한다.

## 절대 원칙 (모든 실행에 적용)
1. **사실 기반.** 확인되지 않은 수치·인물·사건을 만들지 않는다. 불확실하면 "확인 불가"로 명시한다.
2. **본국 출처 우선.** 대상 기업 본국(현지어 포함) 출처를 우선 사용하고, 그다음 국제 신뢰 매체·공시·학술자료를 쓴다.
3. **출처 표기.** 모든 핵심 사실(특히 재무·설립연도·본사)에 출처 URL을 남긴다. 직접 인용보다 요약·정리를 우선한다.
4. **재무필터 통과 기업만 본문에 채택**한다(아래 규칙).
5. **중립·객관.** 과장·홍보성 표현을 쓰지 않는다.
6. **출력 언어는 한국어.** 기업명·지명·전문용어는 원어를 병기한다.

## 재무필터 (품질 게이트 — 가장 중요)
후보 기업은 다음 중 **최소 하나를 출처 URL과 함께** 충족해야 본문에 채택한다.
- (a) 매출/매출액(turnover) 수치 + 출처
- (b) 공식 등기·공시(예: 한국 DART, 인도 MCA, 영국 Companies House 등) + 출처
- (c) 상업 운영 매출을 동반한 투자유치 + 출처
- (d) 증권거래소 상장 사실 + 출처

규칙:
- 위 근거가 없으면 본문에서 제외하고, 리포트 말미 "검증 필요" 목록에 기업명+사유만 남긴다.
- 재무근거 강도를 **[강] / [보통]** 으로 태그한다. **[약]은 본문에서 제외**한다.
- "투자유치만 있고 매출 근거가 없는" 경우는 [약]으로 간주해 제외한다.

## 필수 필드 (채택 기업마다)
- 설립연도 / 본사 위치(도시·국가)
- 핵심 사업
- 주요 파트너
- 지역 유형자원·무형자원과 기술의 융합 방식
- 재무성과(+출처 URL, 강도 태그)

확인 불가한 필드는 임의로 채우지 말고 "확인 불가"로 둔다.

## 입출력 규약
- 상태·아카이브는 **Google Drive 폴더 `place-scan/`** 에 둔다.
  - `seen.json` : 이미 다룬 도시·기업 누적(중복 방지). 매 실행 시작에 읽고, 종료 시 갱신.
  - `reports/YYYY-MM-DD.md` : 당일 리포트 저장.
- 발송 : **Gmail 커넥터**로 운영자 본인 주소에 발송.
- 조사 : **`insane-search`를 기본 도구로** 사용한다. 사용 불가 환경이면 **내장 웹검색/페치로 폴백**한다.
- 문구 다듬기 : 초안 작성 후 **한글 윤문 교정 스킬 `/yunmun`을 반드시 적용**해 AI 번역투·어색한 표현을 교정한다. 문체 기준은 `STYLE.md`.
