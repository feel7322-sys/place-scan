# /place-scan

오늘의 place-based · 브리콜라주 · frugal 혁신 창업기업 **1~2건**을 발굴하고,
`CLAUDE.md`의 재무필터를 통과한 기업만 아래 템플릿으로 출력한다.

## 입력
`$ARGUMENTS`
- 선택적 권역 힌트 (예: "라틴아메리카", "동남아시아 항만도시", "북유럽 폐기물").
- 제외 목록 (seen.json의 도시·기업) — 오케스트레이터(루틴)가 주입한다.

## 절차
1. 권역 힌트가 있으면 그 안에서, 없으면 **아직 다루지 않은 대륙·도시**를 우선해 후보를 찾는다.
   **`CLAUDE.md`의 지역성 필터를 먼저 적용**한다: 수도·수도권, 최대 경제도시, 혁신 허브
   (실리콘밸리·런던·도쿄·서울권 등) 소재 기업은 후보에서 제외하고, **비허브 지방 도시
   2~4곳을 먼저 정한 뒤 그 도시명(현지어 병용)으로 검색**한다. "{국가} startup"식 검색 금지.
2. 제외 목록의 도시·기업은 다루지 않는다.
3. **`insane-search`로** 후보별 사실을 수집한다(미가용 시 내장 웹검색 폴백). **본국(현지어 포함) 출처를 우선**한다.
4. 후보별 필수 필드를 채운다(자원 유형/설립/본사/핵심사업/파트너/자원-기술 재조합/재무성과).
   **`RESOURCE_TYPES.md`의 4유형(①자연·②레거시·③지역문제·④무형) 중 하나(주 유형)로 반드시 태깅**한다.
   개도국 frugal에 한정하지 말고 선진국·개도국을 폭넓게 본다.
   확인 불가 항목은 임의로 채우지 말고 "확인 불가"로 둔다.
5. `CLAUDE.md` 재무필터를 적용한다: (a)~(d) 중 **출처 URL을 동반해 충족한 것만** 본문 채택.
   강도 **[강]/[보통]만 채택**, [약]은 제외.
   성장단계 우선순위도 지킨다: **비상장 성장기업(지역 소재 + 매출 확인)이 기본**,
   상장·유니콘은 본사가 그 지역에 남아 있을 때만 가점. 상장을 '선정 이유'로 삼지 않는다.
6. 통과 기업 **1~2건만** `STYLE.md` 문체로 작성한 뒤, **`/yunmun` 윤문 교정 스킬을 적용**해 한글 표현을 다듬는다.
   채택분을 합쳐 **서로 다른 자원 유형이 2개 이상** 담기게 한다(한 도시 2건이든 두 도시 1건씩이든 무방).
   재무필터 통과가 1건뿐이면 그 1건만 채택하고 유형이 1개인 사유를 한 줄 남긴다(필터 우선).
7. 통과 못 한 후보는 "검증 필요" 목록에 **기업명+사유만** 기록한다.
8. 이번에 다룬 도시·기업을 "신규 추가" 블록에 적는다(seen.json 갱신용).

## 출력 템플릿
**가독성 위주 카드+표 형식으로 쓴다**(불릿 나열 금지). 기업마다 헤더 배지 한 줄 + `| 항목 | 내용 |` 표.

```
# 📍 Place-Based Scan — {YYYY-MM-DD}

**대상 권역:** {권역/도시}
**오늘의 요약:** 채택 **{N}건** · 검증 필요 **{M}건**

---

## {기업명(원어)} — {도시, 국가}
`📍 {도시, 국가}`  `🏷 유형 {①자연/②레거시/③지역문제/④무형}`  `💰 재무 [강|보통]`

| 항목 | 내용 |
|------|------|
| **설립·본사** | {연도} · {도시, 국가} |
| **핵심 사업** | {…} |
| **자원 × 기술** | {어느 자원을 어떤 지식으로 재해석했는지} |
| **주요 파트너** | {…} |
| **재무성과** | [강\|보통] {매출/공시/상장 등} (출처: {URL}) |
| **성장·브리징** | (선택) 장소착근·브리징 / scale-deep↔up / 지식매개 |

---

{필요 시 2번째 기업 동일 형식 + 구분선 ---}

## ⚠️ 검증 필요 (본문 제외)
- **{기업명}** ({도시}) `[{유형}]` — {제외 사유: 매출 근거 없음, 투자유치만 확인 등}

### 출처
- {URL 목록}

### 신규 추가 (seen 갱신용)
- **도시:** [{…}]
- **기업:** [{…}]
```

> 표 안에서 `|` 문자는 `\|`로 이스케이프한다(예: 재무 `[강\|보통]`). 이메일 본문도 같은 카드 구조의 HTML로 만든다.
