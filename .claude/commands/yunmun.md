# /yunmun — 한글 윤문 교정

입력으로 받은 초안 텍스트(또는 직전에 작성한 리포트)의 **한글 표현만** 다듬는다.
**사실·수치·연도·출처·구조는 바꾸지 않는다.** 표현만 자연스러운 한국어로 교정한다.

## 입력
`$ARGUMENTS` — 교정할 텍스트. 없으면 직전에 생성한 리포트를 대상으로 한다.

## 교정 규칙 (AI 번역투 제거)
1. **번역투 어순·표현을 자연스러운 한국어로**
   - "~에 대해/대한"의 남발 → 조사로 풀기 (예: "X에 대해 분석한다" → "X를 분석한다").
   - 불필요한 피동 → 능동 (예: "~로 여겨진다" → 사실이면 "~다", 아니면 "~로 본다").
   - 과도한 명사화 → 동사로 (예: "성장의 실현" → "성장을 이뤘다").
2. **군더더기 제거**
   - "~하는 것을 통해" → "~해서 / ~로".
   - "~라고 할 수 있다", "~인 것으로 보인다" 같은 회피 표현은 사실이면 단정한다.
   - 한 문장 안의 중복 어휘·접속을 정리한다.
3. **기계적 병렬·상투구 제거**
   - "혁신적인, 놀라운, 획기적인" 같은 홍보 형용사 삭제.
   - "결론적으로, 요컨대, 종합하면"의 남발 금지.
4. **외래어·영어 직역 정리**
   - 우리말로 자연스러운 것은 우리말로(예: "솔루션" → "해법", 문맥에 맞게).
   - 단 기업명·고유명사·전문용어는 **원어 병기 유지**(STYLE.md).
5. **문장 길이** — 한 문장에 한 가지 정보. 지나치게 긴 문장은 끊는다.

## 금지
- 사실·수치·연도·출처 URL **변경 금지**.
- 새로운 주장·결론 **추가 금지**(표현만 교정).
- 필수 필드 라벨 구조 **훼손 금지**.

## 출력
교정된 텍스트 **전문만** 출력한다(변경 사유 설명 없이).
