# 루틴(리모트/클라우드) 설정 가이드

이 파일은 **저장소에 둘 필요는 없는** 설정 메모다. 루틴 생성 시 아래 프롬프트를 사용한다.

## 1) 기본 설정
- **트리거**: schedule (cron)
- **cron**: `0 0 * * *` (UTC) → 매일 **09:00 KST**. cron은 UTC 기준이라 09:00 KST = 00:00 UTC.
- **저장소**: `https://github.com/feel7322-sys/place-scan` (main, 읽기 전용 클론)
- **연결 커넥터**:
  - **Google Drive** (seen 상태 읽기·쓰기, `place-scan/reports/` 저장)
  - **Gmail** (`gmailmcp.googleapis.com/mcp/v1`) — 루틴 생성 시 자동 첨부됨.
    쓰기 도구가 `create_draft`뿐이라 **초안(draft)** 으로 리포트를 만든다(자동 발송 아님).

## 2) 루틴 프롬프트 (그대로 붙여넣기)
> 결과는 Gmail 초안 + Drive `place-scan/reports/{날짜} — {사례}/` 폴더 양쪽에 남는다.

```
당신은 place-scan-agent의 일일 실행기다. 클론된 저장소의 CLAUDE.md,
.claude/commands/place-scan.md, .claude/commands/yunmun.md, STYLE.md를 먼저 읽고 그 규칙을 그대로 따른다.

[가장 중요] 최종 산출물은 '채팅에 출력한 리포트 텍스트'가 아니라 실제로 실행한 도구 호출의
결과(Gmail 초안 + Drive 파일들)다. 리포트를 출력하는 것만으로 종료하지 말고, 아래 4·5·6단계의
도구 호출을 반드시 실제로 실행한 뒤 각 결과 ID(draft id, 폴더/파일 id)를 확인하고 종료한다.
도구를 호출하지 않고 리포트만 출력하고 끝내면 실패다.

절차:
1. Google Drive place-scan/에서 seen 상태를 읽는다(최신본 읽기 방식):
   `seen`으로 시작하는 파일 중 날짜가 가장 최신인 것(예: seen-YYYY-MM-DD.json)을 읽고,
   없으면 시드 seen.json을, 그것도 없으면 빈 목록으로 시작한다.
2. 오늘 요일(KST)로 대상 권역을 정한다(아래 "요일별 권역 로테이션").
   그 권역을 place-scan.md의 권역 힌트로 넣어 절차를 수행한다.
   조사는 insane-search를 기본 사용(미가용 시 내장 웹검색 폴백).
   seen 목록의 도시·기업은 제외한다.
   지역성 필터(CLAUDE.md)를 엄격히 적용한다 — 수도·수도권, 최대 경제도시, 혁신 허브
   (실리콘밸리·런던·도쿄·서울권 등) 소재 기업은 제외. 비허브 지방 도시 2~4곳을 먼저 정한 뒤
   그 도시명(현지어 병용)으로 검색한다. 비상장 성장기업(지역 소재+매출 확인)이 기본이고,
   상장·유니콘은 본사가 그 지역에 남아 있을 때만 가점이다(상장 자체는 선정 이유가 아님).
   참신성 필터(CLAUDE.md)를 엄격히 적용한다 — 1990년 이전 설립 기업 제외(기본은 2000년
   이후 설립, 1990~1999년은 예외 사유 명기). ①자연자원 유형은 하루 최대 1건 —
   후보 탐색은 ②레거시·③지역문제·④무형부터 시작한다. 상장기업은 하루 최대 1건
   (창업 지역에 본사가 남은 경우만)이고 비상장 소규모 성장기업이 기본값이다.
   규모는 작아도 된다 — 연매출이 작은 소기업도 출처가 확인되면 채택한다.
   유명 기업도 seen에 없으면 채택 가능(인지도는 배제 사유 아님).
   자연자원 기반 상장사 위주 리포트는 실패다.
   재무필터를 엄격히 적용한다 — 출처 URL 없는 재무근거는 불채택, [약]은 제외.
   기업마다 자원 유형을 주 유형 1개로 태깅하고, 리포트 전체가 서로 다른 유형 2개 이상을
   담도록 한다(한 도시 2건이든 두 도시 1건씩이든 무방). 보통 하루 2건을 목표로 하되,
   재무필터 통과가 1건뿐이면 그 1건만 채택하고 사유를 한 줄 남긴다(필터 우선).
3. STYLE.md 문체로 쓴 뒤 /yunmun 윤문 교정 스킬을 적용해 한글 표현을 다듬어
   최종 한국어 리포트를 완성한다.
4. Gmail로 리포트를 보낸다(수신: feel7322@gmail.com).
   - 제목: [Place-Based Scan] {오늘 날짜} — {기업1(도시1)·기업2(도시2)}
   - 본문: **가독성 HTML 카드형** — 상단 요약(채택 N건/검증 N건) + 기업별 카드
     (위치·유형·재무강도 배지, 라벨/값 표, 출처 링크) + 검증 필요 목록. 홍보성 형용사 금지.
   - Gmail 커넥터가 발송 도구 없이 초안 생성(create_draft)만 지원하면 **초안(draft)** 으로 만든다.
     (초안이어도 무방 — 운영자 계정의 Apps Script(`tools/apps_script_auto_sender.gs`)가
     `[Place-Based Scan]` 제목 초안을 주기적으로 자동 발송한다. 제목 접두어를 바꾸지 말 것.)
5. Google Drive place-scan/reports/ 아래에 **오늘 폴더를 새로 만든다**:
   폴더명 = "{오늘 날짜} — {기업1(도시1)·기업2(도시2)}"
   (예: 2026-07-01 — Kerecis(이사피외르뒤르)·Tillamook(틸라묵)).
   그 폴더 안에 리포트를 **두 형식으로** 저장한다:
   - `{오늘 날짜}.md` (마크다운)
   - `{오늘 날짜}.docx` (MS Word) — 저장소 `tools/md_to_docx.py`로 변환한다:
     `python -m pip install --quiet python-docx` 후
     `python tools/md_to_docx.py {오늘 날짜}.md {오늘 날짜}.docx`, 생성된 .docx를 base64로
     Drive 업로드(contentMimeType=application/vnd.openxmlformats-officedocument.wordprocessingml.document,
     disableConversionToGoogleType=true). 변환이 안 되면 같은 내용을 Google 문서로 저장한다.
6. 이번에 다룬 도시·기업을 직전 목록에 병합해 place-scan/seen-{오늘 날짜}.json을
   **새로 생성**한다(기존 파일 덮어쓰기 아님 — Drive 커넥터에 update/delete 없음).
7. 재무필터를 통과한 기업이 0건이면: 메일 제목과 폴더명 끝에 "(검증 필요만)"을 붙이고
   검증 필요 목록만 담으며, seen는 갱신하지 않는다.

제약: 사실만 쓴다. 미확인 수치·인물 생성 금지. 불확실은 "확인 불가"로 명시.
모든 핵심 사실에 출처 URL을 남긴다. 본국 출처를 우선한다.

요일별 권역 로테이션(KST 기준 오늘 요일로 결정):
- 월: 동남아시아
- 화: 유럽
- 수: 북미
- 목: 남미
- 금: 아프리카
- 토: 중국 또는 일본 — 오늘의 ISO 주차(week number)가 홀수면 중국, 짝수면 일본(격주 교대)
- 일: 한국
정한 권역 안에서 아직 다루지 않은 도시를 우선한다. 해당 권역에서 재무필터 통과 후보를
찾기 어려우면, 같은 권역의 다른 도시를 넓혀 찾는다.
```

## 2.5) 요일별 권역 로테이션 (요약)
| 요일 | 권역 |
|------|------|
| 월 | 동남아시아 |
| 화 | 유럽 |
| 수 | 북미 |
| 목 | 남미 |
| 금 | 아프리카 |
| 토 | 중국 ⇄ 일본 (ISO 주차 홀수=중국 / 짝수=일본, 격주) |
| 일 | 한국 (고정) |

- 기업마다 자원 유형은 **주 유형 1개**로 태깅.
- 하루 리포트는 채택분을 합쳐 **서로 다른 유형 2개 이상**을 담는다(한 도시 2건 / 두 도시 1건씩 모두 가능).
- 보통 하루 2건 목표. 재무필터 통과가 1건뿐이면 그 1건만 채택하고 사유를 남긴다.
- **참신성:** 1990년 이전 설립 제외(기본 2000년 이후), ①자연자원 하루 최대 1건
  (탐색은 ②③④부터), 상장기업 하루 최대 1건. 규모는 작아도 됨(소기업 매출도 출처만 있으면 채택).
  유명 기업도 seen에 없으면 채택 가능 — 인지도는 배제 사유가 아님.

## 3) 활성화 전 점검 (필수)
1. `state-seed/seen.json`을 Google Drive `place-scan/` 폴더에 업로드한다.
2. 루틴을 만들되, **먼저 "Run Now"(또는 `claude routines run <이름>`)로 1회 수동 실행**한다.
3. 확인 항목:
   - [ ] 오늘 요일에 맞는 권역이 선택됐는가
   - [ ] Gmail 초안이 가독성 HTML로 생성됐는가
   - [ ] Drive `reports/{날짜} — {사례}/` 폴더가 생성되고 리포트가 저장됐는가
   - [ ] insane-search가 실제로 호출되는가(안 되면 폴백 동작 확인)
   - [ ] /yunmun 교정이 적용돼 한글이 자연스러운가
   - [ ] 모든 재무 사실에 출처 URL이 붙는가
   - [ ] 재무필터가 동작하는가(근거 약한 기업이 본문에 안 들어가는가)
   - [ ] 참신성 필터가 동작하는가(1990년 이전 설립 없음, ①자연자원·상장기업 각 최대 1건)
   - [ ] 하루 리포트에 서로 다른 자원 유형이 2개 이상 담겼는가
   - [ ] `seen-{날짜}.json`이 새로 생성되는가(최신본 읽기)
   - [ ] 시드에 있는 4개 기업이 반복되지 않는가
4. 이상 없으면 스케줄을 활성화한다.

## 4) 주의
- 루틴은 **리서치 프리뷰**다 — 한도·동작이 바뀔 수 있다.
- **일일 실행 상한**은 plan별로 있으며 공식 수치는 비공개다. `claude.ai/settings/usage`에서 확인한다.
- 매 실행이 저장소를 **새로 클론**하므로, 상태는 반드시 Drive에 쓴다(저장소에 쓰지 않는다).
- **insane-search 가용성**: 로컬 플러그인이라 클라우드 루틴에서 인식 안 될 수 있다.
  "Run Now" 테스트 때 실제 호출 여부를 확인하고, 안 되면 내장 웹검색 폴백으로 동작한다.
  (`/yunmun`은 repo 커밋 스킬이라 클라우드에서도 동작한다.)
